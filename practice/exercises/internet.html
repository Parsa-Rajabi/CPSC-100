  5. Which of the following is NOT a component of URL's?

    1. File & Directory
    2. Domain
    3. Server
    4. Protocol

    C

    Protocols specify how the client and server should communicate with each other. For example, when you use the Internet, you may have noticed that some sites you visit use "http" at the beginning of the URL while others use "https". HTTP stands for hypertext transport protocol and it is a set of standards that dictate how the client and server should communicate in order for the client to fetch files from the server and for the server to obtain input from the client.

    Domain names allow people to easily identify a website. An example of a domain name is "google.ca".

    File & directory is a way to specify where a file lives on the server. For example, "/course/main.php" indicates that a file (main.php) lives in a folder called course.

    A server is a machine that provides a service for other machines (called clients). For example, a server may store files for other machines to access or it may be responsible for performing computations for other machines.

  7. At any place in which two or more networks intersect one another, there must be a device called a...?
    1. Server
    2. Client
    3. ISP
    4. Router
    5. None of the above

    D.

    Routers act as the supervisor for the transfer of data from one network to another. Conceptualize a router as a gatekeeper, or doorway, that facilitates the transfer of information. For example, in order for data on your personal computer to reach your ISP (Internet Service Provider), a router must act as the mediator in conducting the transfer of that data. Therefore, wherever two networks intersect, there must be a router acting as the facilitator of the connection between those networks.

  9. Which of the following is correct about Transmission Control Protocol/Internet Protocol (TCP/IP), a.k.a. how computers send information to each other?
    1. The IP packets that are being sent are dependent on one another
    2. The protocol can throw away packets if there is congestion
    3. The IP packets are sent at the same time
    4. All of the above

    A is incorrect. The IP packets are sent independently to avoid the delay of transmissions (page 51).

    B is correct. If the protocol detects slow progress of a packet, it will throw it away to decrease traffic. When the packet is not quickly detected at the destination, the recipient server will request for it to be re-sent. (page 51).

    C is incorrect. Packets are sent independently and one at a time (page 50).

    D is incorrect as not all of the above answers are correct.
